# restaurante_app
La aplicación busca simular el proceso básico en un restaurante, permitiendo el registro de productos y clientes, donde se puede visualizar luego esta información.
El sistema se estructuró como fue solicitado para la práctica, en la carpeta modelos se encuentran las clases cliente y producto. La carpeta servicios contiene a la clase restaurante que es la lógica principal. Por último está el main.py que es donde todo ocurre, el registro, almacenamiento y muestra de la información.
Gracias a que está separado por partes es facilmente un trabajo en colaboración con otras personas, ya que los cambios de uno no necesariamente entorpecerán el avance de los demás.
